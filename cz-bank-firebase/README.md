# CZ Bank（Firebase版）

OCコミュニティ向けの仮想通貨管理サイトです。React + Vite + Firebase（Authentication / Firestore）で作られており、複数人のメンバーで残高・取引履歴・入出金申請がリアルタイムに共有されます。

## 公開までの流れ（全部で6ステップ）

### ステップ0：Googleアカウントを作る

お持ちでないとのことなので、まず https://accounts.google.com/signup で無料のGoogleアカウントを作成してください（普段使いのメールアドレスで構いません）。

### ステップ1：Firebaseプロジェクトを作る

1. https://console.firebase.google.com/ を開き、Googleアカウントでログイン
2. 「プロジェクトを作成」→ 好きなプロジェクト名（例：cz-bank）を入力 → 「続行」
3. Googleアナリティクスは今回不要なので、オフのままで「プロジェクトを作成」

### ステップ2：Firestore（データベース）を有効にする

1. 左メニューの「構築」→「Firestore Database」を開く
2. 「データベースの作成」→ ロケーションは `asia-northeast1（東京）` を選択
3. セキュリティルールは「本番環境モード」を選択（あとで書き換えます）

### ステップ3：Authentication（ログイン機能）を有効にする

1. 左メニューの「構築」→「Authentication」を開く
2. 「始める」→ ログイン方法の一覧から「メール/パスワード」を選び、有効にする

### ステップ4：Webアプリを登録して接続情報（config）を取得する

1. プロジェクトのトップ画面（歯車アイコン→「プロジェクトの設定」）→ 下の方の「マイアプリ」→ `</>`（ウェブ）のアイコンをクリック
2. アプリ名を適当に入力（例：cz-bank-web）→ 「アプリを登録」
3. 表示される `firebaseConfig = { apiKey: "...", ... }` の中身をまるごとコピーし、このプロジェクトの `src/firebaseConfig.js` を開いて、`YOUR_API_KEY` などのプレースホルダー部分を実際の値に置き換えて保存してください

### ステップ5：最初の管理者アカウントを手動で作る

ここだけは自動化できないので、Firebaseの管理画面から手作業で1回だけ行います。

1. 「Authentication」→「Users」タブ→「ユーザーを追加」
   - メールアドレス：管理者として使いたいメールアドレス
   - パスワード：`JP009`（後でアプリにログインした後、Firebaseの管理画面や再設定メールで変更可能です）
2. 追加されたユーザーの行に表示されている **ユーザーUID**（英数字の羅列）をコピーしておく
3. 「Firestore Database」→「データ」タブ→「コレクションを開始」
   - コレクションID：`accounts`
   - ドキュメントID：さきほどコピーした **ユーザーUID** を貼り付け
   - フィールドを3つ追加：
     | フィールド名 | 型 | 値 |
     |---|---|---|
     | name | string | 運営局(管理者) |
     | role | string | admin |
     | balance | number | 0 |
4. 同じ `accounts` コレクションに、もう1つドキュメントを追加
   - ドキュメントID：`reserve`（この文字列そのまま）
   - フィールド：
     | フィールド名 | 型 | 値 |
     |---|---|---|
     | name | string | 発行準備金 |
     | role | string | reserve |
     | balance | number | 1000000 |

これでログイン用のアカウント（`admin@...` / `JP009`）と、通貨発行の元になる「発行準備金」口座の準備ができました。

### ステップ6：セキュリティルールを設定する

1. 「Firestore Database」→「ルール」タブを開く
2. このプロジェクト内の `firestore.rules` の中身を全部コピーし、既存の内容を置き換えて「公開」を押す

これで、他のメンバーがブラウザの開発者ツールから不正に自分の残高を書き換えたりできないよう、最低限のガードがかかります。

## GitHub Pagesに公開する

1. GitHubで新しいリポジトリを作成し、このフォルダの中身をすべてプッシュします。

   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<あなたのユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```

2. リポジトリの **Settings → Pages** を開き、Source を **GitHub Actions** に設定
3. `main` にプッシュすると自動でビルド・公開されます（Actionsタブで進行状況を確認できます）
4. 数分後、`https://<あなたのユーザー名>.github.io/<リポジトリ名>/` で公開されます

## ローカルで動作確認する

```bash
npm install
npm run dev
```

## このバージョンでできること

- メールアドレス＋パスワードでの本格的なログイン・新規登録
- 残高・取引履歴・入出金申請は **Firestore（クラウド）に保存**され、全メンバーの画面にリアルタイムで反映されます
- 新規登録したメンバーは残高0からスタートし、管理者が「発行」機能で残高を付与します
- 入金・出金は申請制で、管理者画面から承認/却下できます
- 送金や承認処理はFirestoreの「トランザクション」機能を使っており、複数人が同時に操作しても残高の整合性が壊れないようにしています

## 知っておいていただきたい制限

- セキュリティルールは「コミュニティ内の仮想通貨」として妥当な強度にしていますが、実在するお金や資産を扱う本物の銀行と同じレベルの堅牢さではありません
- 万が一データをリセットしたい場合は、Firestoreの各コレクション（`accounts`・`requests`・`transactions`）のドキュメントを手動で削除してください（`reserve`と管理者の`accounts`ドキュメントは残すことをおすすめします）
